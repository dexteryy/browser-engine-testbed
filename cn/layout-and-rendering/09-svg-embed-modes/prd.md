测试用例 9：SVG 图像嵌入与引用方式

测试目的：验证 Chrome 对 SVG 图形以不同嵌入方式（内联、外部引用、CSS 背景）的渲染支持情况，并比较它们在样式控制和性能上的差异。通过该测试，确保各种 SVG 用法都能正确显示矢量图像，且各自特性如 CSS 可控性、缩放清晰度等符合预期。

测试特性：涵盖三种常见 SVG 使用方式：

- 内联 SVG：直接在 HTML 中嵌入`<svg>`及其子元素。这使 SVG 成为 DOM 的一部分，页面的 CSS 可以选中 SVG 内部元素进行样式调整，SVG 内部也可定义自己的样式。
- 外部 SVG 引用：通过`<img src="image.svg">`加载外部 SVG 文件，或使用`<object>`/`<embed>`标签。这种方式下 SVG 不会展开为 DOM 节点，内部元素无法被页面 CSS 直接选择。
- CSS 背景 SVG：在 CSS 中使用 background-image: url(image.svg)将 SVG 作背景图像应用于某元素。这与类似，SVG 不直接出现在 DOM 中，仅作为图像资源呈现。

供你参考的测试用例设计：页面分为三部分展示上述三种 SVG 嵌入手段：

1. 内联 SVG：HTML 直接包含一个`<svg>`图形（例如绘制一个红色圆形和黑色矩形的简易 SVG），并为其中某元素添加 class（如给圆形 class="dot"）。在页面 CSS 中定义对应类选择器样式（例如.dot { fill: green; }）改变该元素填充色。
2. 外部 SVG 引用：使用`<img src="example.svg" alt="示例SVG">`插入一幅外部 SVG 文件（内容与内联 SVG 相似）。不给特别的 CSS，使其按 SVG 自身的宽高或 viewBox 默认尺寸呈现。
3. CSS 背景 SVG：有一个元素，在其 CSS 中使用 background-image: url(example.svg)设定与上述相同的 SVG 作为背景。通过 width/height 或 background-size 控制背景图显示尺寸。

三种方式的 SVG 图像在视觉上应基本一致，并排展示便于比较。内联 SVG 的圆形由于页面 CSS 作用应呈绿色，而引用的 SVG 保持原来的红色（除非 SVG 内部自带样式修改），背景 SVG 则根据背景属性渲染。

供你参考的测试用例使用方法：同时检查三种 SVG 渲染：确认它们均清晰显示完整的图形。内联 SVG 内部元素受页面 CSS 影响，应发生预期的样式改变（如圆形由红变绿），而外部的 SVG 内容不受页面 CSS 干扰，应保留原样颜色，实现两者差异对比。背景 SVG 应正确地出现在元素背景区域，没有裁剪或拉伸畸形，若设置了 background-size，检查其缩放是否按比例。可使用 DevTools 验证：在元素面板中能直接看到内联 SVG 的 DOM 结构和元素属性，而方式只能看到一个节点（内部 SVG 不可见），背景图则作为 CSS 属性显示。放大页面或者检查高 DPI 下，确认 SVG 图像在各方式下都保持矢量图的清晰度（无像素模糊）。性能方面，如果需要，可比较内联 SVG 与外部 SVG 的差异：例如重复使用相同 SVG 时，外部引用可被浏览器缓存，而多次内联会增加 HTML 大小。在本测试使用的小 SVG 下差异可以忽略，但仍验证 Chrome 对各方法的加载渲染均无异常。确保没有出现如外部 SVG 无法加载、背景 SVG 不显示等问题。
