测试用例 12：CSS 遮罩（Mask）效果及性能

测试目的：验证 Chrome 对 CSS 遮罩属性（mask/mask-image 等）的支持，以及遮罩对元素可见区域和渲染性能的影响。通过该测试，检查使用遮罩后元素的形状裁剪是否符合预期（仅指定的形状区域可见，其余部分透明），并观察 Chrome 是否为应用遮罩的元素创建独立合成层以优化绘制 。

测试特性：使用 CSS mask-image 或简写 mask 属性给元素应用矢量或图像遮罩。例如使用一个 SVG 形状作为遮罩图案，或使用 CSS 渐变作为遮罩，实现非矩形的元素可见区域。测试内容包括：遮罩裁剪的准确性（元素仅在遮罩形状范围内显示）、遮罩的透明渐变效果、以及遮罩与背景的组合显示。关注遮罩对性能的潜在影响，如遮罩元素是否触发 GPU 加速或增加重绘负担。

供你参考的测试用例设计：页面中有一个带背景的块级元素（例如`<div class="avatar">`，内部可放一张照片或纯色填充），通过 CSS 设置其 mask-image 属性引用一张 SVG 遮罩图（例如一个白色圆形图案），并使用 mask-mode: alpha（默认）按 alpha 通道应用遮罩。CSS 调整 mask-size 为与元素等同大小，mask-position: center 使遮罩图案居中覆盖。这样，该元素最终呈现为圆形头像效果：圆形区域显示照片，四角被遮罩隐藏。为了对比，可在其旁边放置相同大小的不遮罩元素作为参考。页面底部背景采用明显不同的颜色，以便清晰看出遮罩隐藏部分是真正透明（能透出背后的背景）。

供你参考的测试用例使用方法：页面加载后，检查遮罩元素的外观：应只显示圆形范围内的内容，外部区域完全透明，使背后的底色显露。圆形边缘应平滑，无锯齿或硬边（SVG 遮罩会使边缘抗锯齿处理）。若遮罩图案含半透明渐变，观察元素边缘呈现渐变淡出效果是否正确。对比未遮罩的参考元素，确认遮罩仅影响目标元素而不影响周围布局。接着在 DevTools 元素面板查看该元素的 CSS 属性，确认 mask-image 等已生效，以及该元素在渲染层次上可能被提升为合成层（使用“Layer Borders”查看，有无橙色边框标识 ），因为 Chrome 对具有遮罩的元素通常会单独合成处理 。滚动或触发页面其他动画（如果有），留意遮罩元素部分是否发生异常重绘。一般静态遮罩不会持续消耗性能，但首次绘制遮罩可能比普通元素略慢，可在 Performance 面板查看该元素首次 Paint 耗时，如在可接受范围则表示性能正常。最后验证各种浏览器行为：例如调整窗口大小或在不同 DPI 屏幕下，遮罩效果保持正确比例；在 Chrome 下未出现遮罩区域错位、内容泄露（遮罩外露出内容）等 bug，一切表现符合预期。
