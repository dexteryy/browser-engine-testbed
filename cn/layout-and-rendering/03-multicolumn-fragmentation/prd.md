测试用例 3：多列布局（Multicolumn）与内容分栏

测试目的：评估 Chrome 对 CSS 多列布局的支持，以及在大量内容分栏时的渲染正确性和性能表现。该测试关注文本和元素在多个列中自动分隔（fragmentation）的效果，检查分页符、列间距等是否符合预期，并观察浏览器在复杂分栏场景下的首次渲染时间和滚动性能。

测试特性：使用 CSS 多栏布局属性，例如 column-count 和 column-width 定义固定列数或列宽，column-gap 设置列间距，以及避免元素跨栏拆分的规则（如 break-inside: avoid 防止元素在列间断开）。包含段落文本、图像等块级内容，以及需要整体保持的元素（例如整段短语或图片完整显示），以测试 Chrome 对内容分割和避断的处理。可选地包含 column-span 属性的元素（如跨越所有列的标题），以测试跨栏元素的布局。

供你参考的测试用例设计：页面中有一个多栏容器（例如宽度固定的<div>`），应用诸如 column-count: 3 或 column-width: 200px 的 CSS，使其内部文字内容自动分成三栏显示。容器内放置长篇连续文本、若干图片和小标题。为某些元素（如每段开头的小标题或整张图片）设置 CSS 属性 break-inside: avoid，以确保它们不会被拆分到不同列。另在容器顶部添加一个横跨多列的标题元素，使用 column-span: all（如果 Chrome 已支持）使其占据整个多栏容器宽度。

供你参考的测试用例使用方法：加载页面后，检查文本是否按照预期分布在多列中，各列高度大致均衡且没有内容缺失。特别关注设置了避免断开的元素：确保它们整体出现在单一列中（如某图片过长无法放入当前列，应自动移至下一列顶部），没有出现半截内容被切断的情况。确认列间距（column-gap）一致且等于设定值，列与列之间的分隔线（如果使用 column-rule 的话）正确渲染。滚动页面，观察在进入新列内容时是否有明显延迟或卡顿。对于性能，可在 DevTools 的 Performance 面板中记录首次渲染时间，特别是在长文本分栏的场景下，观察 Chrome 排版引擎将段落拆分成多列所耗费的时间。若需要，可通过启用绘制闪烁（Paint Flashing）查看滚动时各列内容的重绘情况，以评估多列布局在滚动中的表现 。确认整个多栏布局无排版错误，各列顺序正确（一般从左到右，或根据 writing-mode 方向），视觉上连贯流畅。
