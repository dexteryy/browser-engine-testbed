## 研发侧⽤例 06 ｜ Canvas/Video 互操作：像素拷⻉与纹理上传

### 测试目标

验证 `<video>` 帧拷⻉到 Canvas 2D 与 WebGL 纹理的路径开销、上传成本与⾊彩精度（YUV→RGB 转
换+纹理采样）。

### 涉及特性

`<video>`、Canvas 2D 的 drawImage(video, ...)、WebGL texImage2D/texSubImage2D(video)；CSS
image-rendering 采样差异对照。

### 页面结构与要点

- 左列：Canvas 2D 从 `<video>` 持续绘制。
- 右列：WebGL 以 `<video>` 为纹理渲染到四边形。
- 两列叠加⽹格线与测试图样（⽅便核对缩放与⾊彩）。

### 人工观察与建议工具

- Performance：连续 10s 录制，⽐较两侧主线程与 GPU 负载。
- Layers：确认 WebGL 画布是否独⽴合成、上传带宽是否造成卡顿（间接观察）。
- 屏幕观察：⾊彩/锐度是否与直接 `<video>` 呈现⼀致；⽆撕裂。

### 验收标准

- 两路径视觉⼀致（⾊彩与精度偏差在⾁眼可接受范围）。
- 30/60fps 画⾯⽆周期性掉帧。
- 纹理上传不会导致主线程⻓抖动。

### 可选变体

- ⾼分辨率 4K 源；
- 将 WebGL 纹理再叠加 CSS filter 观察⼆次合成成本。
