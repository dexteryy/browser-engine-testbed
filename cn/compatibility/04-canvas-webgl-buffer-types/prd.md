## 研发侧⽤例 04 ｜ Canvas/WebGL 内部缓冲类型覆盖

### 测试目标

覆盖 Canvas 2D 与 WebGL(1/2) 在不同缓冲配置下的底层资源与光栅路径差异（含是否保留缓冲、
alpha 预乘、抗锯⻮、多重采样）。

### 涉及特性

- Canvas 2D：alpha: true/false、willReadFrequently（谨慎）
- WebGL：preserveDrawingBuffer: true/false、premultipliedAlpha: true/false、antialias:
  true/false、WebGL2 的 MSAA/FBO 变体。

### 页面结构与要点

- 以 2×2 ⽹格同时展⽰ 4 组配置（⽰例：alpha×preserve 的四种组合），统⼀渲染相同图元（半
  透明渐变+⽂本+细线）。
- 另建⻚⾯覆盖 WebGL2 MSAA FBO 与普通 backbuffer 的对⽐。

### 人工观察与建议工具

- Layers：观察是否单独合成层、缓冲⼤⼩。

- Performance：查看 Raster 与 Composite；切换 preserveDrawingBuffer 时对合成与拷⻉成本的
  影响。
- Paint flashing：验证 2D Canvas 在更新时的重绘范围。

### 验收标准

- 视觉上预乘与⾮预乘的边缘、渐变过渡符合预期。
- 关闭 antialias 能看到线条锯⻮⽽不引⼊异常重采样或抖动。
- preserveDrawingBuffer 不造成异常内存暴涨。

### 可选变体

- 放⼤画布到全屏，观察⽡⽚栅格化压⼒。
- 叠加 CSS filter 到 Canvas 元素，迫使额外 render pass。
