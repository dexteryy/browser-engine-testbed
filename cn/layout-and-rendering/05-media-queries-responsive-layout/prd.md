测试用例 5：媒体查询响应式布局

测试目的：验证 Chrome 对于 CSS 媒体查询（Media Queries）的支持，以及在不同视口尺寸下应用不同样式的响应式布局能否正确渲染。通过该测试，可确保页面在宽屏与窄屏设备上根据断点切换布局时没有布局错乱或闪烁，并观察布局切换时的性能表现。

测试特性：使用@media 媒体查询定义多个断点样式。例如针对不同视窗宽度（如 min-width 或 max-width 条件）切换 CSS：在桌面宽度时采用多栏布局、较大间距和导航横排显示；在移动宽度时采用单栏堆叠布局、更大的字体和垂直导航等。涉及的 CSS 特性包括布局调整（如改变 display 类型或 flex-direction）、元素显隐（使用 display: none 在特定断点隐藏某些元素）、排版调整（如字体大小改变）等。完全依赖纯 CSS 媒体查询实现响应式，无需 JavaScript 参与。

供你参考的测试用例设计：页面包含头部导航、主内容区和侧边栏三个部分。通过 CSS 媒体查询，在视口宽度大于 1200px 时，将页面设为两栏布局（侧边栏和主内容并排，导航菜单横向展开）；当宽度小于 800px 时，切换为单栏布局（侧边栏下移或隐藏，导航变为垂直列表）。中间宽度范围可以设置另一个样式（如平板设备下侧边栏显示但导航仍保持横排）。不同断点下页面元素会调整布局和样式，例如窄屏时隐藏某些次要板块、导航菜单收起为汉堡按钮等。

供你参考的测试用例使用方法：在 Chrome 中逐渐缩放窗口宽度或使用 DevTools 的设备模拟器，观察页面布局在预设断点处是否发生预期的变化。关注断点切换瞬间是否平稳：样式改变应立即生效，布局重新流动过程中没有明显闪烁或内容重叠。验证各断点下的布局细节：如窄屏状态侧边栏是否确实隐藏或移至内容下方，导航菜单是否切换为垂直列表显示。检查图片、文字等是否根据新的容器宽度自动调整尺寸或换行。性能方面，可记录窗口 resize 过程中浏览器的表现：频繁拖动窗口时，Chrome 应高效应用媒体查询样式，不应出现卡顿或崩溃。也可通过监测 CLS（累计布局偏移）指标确保布局切换对视觉稳定性影响最小。确认不同屏幕尺寸下页面布局均正常，没有元素错位、溢出或不可见的情况。
