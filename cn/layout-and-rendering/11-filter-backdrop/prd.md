测试用例 11：滤镜效果与背景滤镜性能

测试目的：评估 Chrome 对 CSS 滤镜（filter）及背景滤镜（backdrop-filter）效果的渲染准确性，并关注这些效果对页面重绘和性能的影响。该测试特别针对模糊滤镜（blur）等高开销效果在浏览器中的表现：验证前景元素滤镜和背景滤镜的视觉效果正确，同时观察其是否引发频繁重绘或帧率下降 。

测试特性：使用 CSS filter 属性对元素应用图像效果（例如 filter: blur(5px)模糊），以及使用 backdrop-filter 对半透明遮罩元素后的背景应用效果（例如 backdrop-filter: blur(5px)）。测试场景包括：一个内容较长可滚动的页面以及一个固定在顶部的半透明栏，设置 backdrop-filter: blur(5px)使滚动时经过其后的内容被模糊；正文中插入一张图或色块元素，应用 filter: blur(5px)使其自身模糊。滤镜强度设为中等以便观察明显效果。

供你参考的测试用例设计：页面头部有一个固定定位的，样式为半透明背景（例如 background: rgba(255,255,255,0.8)）并添加 backdrop-filter: blur(5px)。主体部分包含足够长的文本段落列表以产生滚动，并在中部插入一张图片`<img class="photo">`，通过 CSS 设置 filter: blur(5px)。页面加载时 header 遮盖顶部部分内容，用户可以滚动页面让不同内容经过 header 后方，从而观察背景模糊效果；同时正文中那张图片无论是否在视口，都一直应用模糊滤镜。

供你参考的测试用例使用方法：首先静止状态检查：固定头部区域应呈现出明显的模糊背景效果（类似毛玻璃），其下的文字图片透过它变得虚化 ；正文中图片应肉眼可见模糊，边缘细节模糊不清，验证 filter 生效。然后滚动页面，关注两方面：一是背景内容在经过 header 遮罩时模糊效果是否实时更新且平滑，没有出现未模糊的瞬间；二是页面滚动流畅度，留意是否因为应用滤镜导致帧率下降或滚动卡顿。如需量化，可开启 DevTools 的“Paint Flashing”（重绘区域高亮） ：若发现 header 区域在滚动过程中每帧都闪烁绿色，表示每帧都在重绘背景，说明 backdrop-filter 引发大量重绘。图片本身由于始终静止，应用滤镜后一般会被提升到 GPU 合成层以避免重复绘制，但若未自动提升，可观察其在滚动时是否也反复重绘。可进一步打开“Layer Borders”检查：理想情况下，使用 backdrop-filter 的 header 和使用 filter 的图片都会各自成为一独立合成层 ，这样滚动时背景重绘和滤镜计算在 GPU 完成，减轻 CPU 压力。性能方面，在 DevTools Performance 录制滚动，可以关注 Paint 和 Composite 阶段耗时；模糊滤镜属于较重的计算，每增加 1px 模糊半径需要考虑相邻像素 ，在低端设备可能成为瓶颈。如果观察到帧率明显下降，可尝试对相应元素使用 will-change: transform 提示浏览器优化。最后确认：滤镜效果视觉正确，Chrome 对前景滤镜和背景滤镜都能正确渲染，没有出现区域未模糊或错误遮罩的情况；在合理优化下，滚动和交互依然流畅。
