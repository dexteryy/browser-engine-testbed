测试用例 10：混合模式叠加效果与合成层

测试目的：验证 Chrome 对 CSS 混合模式（mix-blend-mode）的渲染支持和图层处理，包括元素与背景内容的混合显示是否正确，以及混合模式对页面合成层的影响 。通过该测试，可以确认不同混合模式（如 Multiply、Screen 等）的视觉效果符合预期，同时观察浏览器是否为混合元素创建适当的合成图层以保证性能。

测试特性：使用 CSS mix-blend-mode 属性将一个元素的内容与其背景进行特定混合模式的叠加显示。例如测试 mix-blend-mode: multiply（正片叠底）或 screen（滤色）等典型模式。页面包含重叠的元素来产生混合效果：底层元素设置纯色背景，上层元素设置半透明颜色或包含图形，并应用非 normal 的 blend 模式。还可涉及 isolation 属性来测试隔离对混合结果的影响（可选）。通过不同混合模式（对文本、图像等）的组合测试 Chrome 是否正确实现各混合公式。

供你参考的测试用例设计：页面有一个大块背景元素（例如`<div class="bg">`）填充纯蓝色。其上方部分区域放置一个较小的，背景设置半透明红色（例如 background: rgba(255,0,0,0.5)），并应用 mix-blend-mode: multiply。这样两者重叠区域应呈现蓝红正片叠底后的颜色（预期偏紫色），未重叠部分 blend-box 依然显示半透明红叠加在页面底色上。可在 blend-box 内再放置一些文字以观察文字与背景的混合（如白色文字在 multiply 模式下会变色融合到底色）。如果需要，也可增加一个元素设置 isolation: isolate 来验证混合隔离效果。

供你参考的测试用例使用方法：首先确认视觉混合效果：blend-box 与底层重叠部分颜色应为预期的混合色，未叠加部分仍保持自身半透明红。混合边界应平滑，没有出现异常的色带或闪烁。接着使用 DevTools 的渲染工具，启用“Layer Borders”选项，观察 blend-box 元素是否被提升为单独的合成图层（其边框应出现橙色/橄榄色边线 ），因为具有非正常混合模式的元素会创建新的绘制层 。然后留意性能：由于混合计算通常由 GPU 在合成阶段完成，单个混合元素对帧率影响很小。可在 Performance 面板录制滚动或内容变化时的帧率，验证即使存在混合模式，页面重绘开销未显著增加。如需测试极端情况，可增大 blend-box 面积或数量，观察是否出现帧率下降。总体验证 Chrome 对 mix-blend-mode 支持良好，叠加渲染正确，无错位或顺序错误，性能无明显损耗。
