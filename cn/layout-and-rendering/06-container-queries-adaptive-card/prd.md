测试用例 6：容器查询（Container Queries）自适应组件

测试目的：验证 Chrome 对 CSS 容器查询的支持，以及组件根据其父容器尺寸自适应布局的能力 。不同于媒体查询针对整个视口，容器查询允许在同一页面中，不同区域的组件各自响应其所在容器大小，实现更精细的“微响应式”设计。通过此测试，确保 Chrome 正确应用容器查询规则，使组件在各种容器上下文中都能呈现合适布局，并观察在窗口/容器尺寸变化时样式切换的性能。

测试特性：使用 CSS 容器查询语法，包括在容器元素上设置 container-type（如 inline-size）和 container-name，以及@container 规则内部编写条件样式。例如，当容器宽度达到某一阈值时，改变子元素的排列或样式。测试涉及现代 CSS 功能，如在组件内部通过容器查询切换 Flex 布局方向、调整字体大小或显示/隐藏某些元素。强调容器查询相对于媒体查询的独立性：页面中可同时存在多个容器，各自达到条件时触发不同的样式，以验证其互不干扰的工作方式。

供你参考的测试用例设计：页面包含两个功能相同但尺寸不同的组件实例（例如“卡片 card”组件），分别置于大小不同的容器中。一种容器为宽幅区域（比如主内容区，宽度较大），另一种为窄幅区域（比如侧栏，宽度较小）。每个组件内部都有布局可以根据容器大小调整，比如包含一张图片和一些文本描述：在宽容器中采用左右并排布局（图片和文本并排），在窄容器中改为上下堆叠布局（图片在上，文字在下）。通过给容器设置 container-name 并在组件 CSS 中使用@container 规则检测容器宽度，定义阈值（例如 400px）以上用横向布局，以下用纵向布局。这样在同一视口下，大容器内的卡片将呈横排布局，而小容器内的卡片呈竖排布局。

供你参考的测试用例使用方法：打开页面后，同时观察两个容器内组件的呈现：确认它们根据各自容器的大小显示出不同布局（大容器内的组件为横向布局，小容器内的组件为纵向布局），符合容器查询的预期效果。然后手动调整浏览器窗口宽度，注意当容器尺寸跨过容器查询定义的阈值时，相应组件的布局是否随之改变（例如当窗口变窄导致原本宽容器变小时，内部卡片从横排切换为竖排）。这一切应当独立于页面整体的媒体查询；即无需整个视口达到某宽度，只要某个容器大小达到条件，其内部组件就会自行切换样式。可在 DevTools 元素面板查看容器的实际尺寸和命名，以及对应的@container 规则是否被触发应用。性能观察方面，在调整窗口尺寸导致容器查询触发时，Chrome 样式重计算应相对局部化、效率高。使用 Performance 工具记录容器尺寸变化过程，检查样式计算开销，并留意是否有明显卡顿。确认容器查询在 Chrome 中工作稳定，没有出现应匹配样式未应用或错误应用的情况。
