测试用例 4：垂直书写模式与文本方向

测试目的：验证 Chrome 在垂直书写模式（vertical writing mode）下的排版表现，包括文本方向、字符旋转和布局定位等方面的正确性。通过该测试，可以检查不同语言（如中英文混排）在垂直模式下的渲染是否符合预期，并发现浏览器在 vertical-rl 等模式下有无兼容性问题或排版错误。

测试特性：使用 CSS 属性 writing-mode 将容器设置为垂直排版，例如 writing-mode: vertical-rl（文字从上到下、行从右到左）或 vertical-lr。结合 text-orientation 属性控制字符方向（如默认的 mixed 使西文字符旋转，东亚字符直立）。页面中混合使用中日韩文字（其在垂直排版中应保持竖直显示）和拉丁字母（应旋转 90 度横排显示），以及标点符号、数字等，测试其在垂直排版下的位置和形态。还可包含一个采用垂直书写模式的容器并配合其他布局（如该容器本身处于 Flex 或 Grid 布局中）以测试不同书写模式之间的对齐关系。

供你参考的测试用例设计：一个主要测试区域被设置 writing-mode: vertical-rl，其中包含一段中英文混杂的文本。例如，段落中含有中文句子、英文单词、数字编号及括号等符号。可以加入换行和不同块级元素（如`<p>`段落和`<ul>`列表）以观察在垂直模式下的排列。旁边放置一个普通水平书写模式的文本区域，方便对比两种模式的差异。在垂直文本区域中，针对英文单词可以应用 CSS 属性 text-orientation: upright 或 sideways 进行测试，看其效果差异。

供你参考的测试用例使用方法：目视检查垂直排版区域内的文本流向是否正确：文字应从上往下排列，段落顺序从右向左推进（对于 vertical-rl 模式）。汉字等东亚字符应直立显示，英文等拉丁字符应垂直旋转（除非使用 text-orientation: upright 强制直立）。检查标点符号的位置，例如句号、逗号在垂直排版中应位于字符右下角或经过适当旋转，Chrome 应正确处理这些细节。观察不同元素的对齐：比如多行段落在垂直方向上是否顶端对齐齐整，列表项目符号在垂直模式下的位置是否正常。将页面滚动或调整窗口大小，确保垂直书写的内容没有出现错位或重叠。如果垂直容器嵌在其他布局中，检查 Chrome 是否正确计算该容器的尺寸和位置（Chrome 的 LayoutNG 对垂直书写模式提供了更好的支持，修复了许多垂直模式下的定位问题 ）。确认整个垂直文本区域可正常阅读，没有字符渲染方向错误或被裁剪遮挡的问题。
