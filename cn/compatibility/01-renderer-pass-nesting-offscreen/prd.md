## 研发侧⽤例 01 ｜ Renderer Pass 嵌套与离屏缓冲链路

### 测试目标

验证在存在多层 filter/backdrop-filter/mask/混合模式的情况下，Chrome 是否为各层正确创建并复
⽤离屏缓冲（offscreen render pass），合成顺序与边界是否正确；评估多级嵌套对光栅化与显存的
影响。

### 涉及特性

CSS filter: blur()/drop-shadow()、backdrop-filter: blur()、mask-image/-webkit-mask-image、
mix-blend-mode、isolation: isolate、半透明叠加与不透明内容混排；可加⼊ SVG filter 版本做对
照。

### 页面结构与要点

- 三层嵌套容器：外层 A（filter: blur(6px)），中层 B（mix-blend-mode: multiply + 半透明背
  景），内层 C（mask-image 圆形或 SVG 遮罩）。
- 额外覆盖层 O（position: fixed; backdrop-filter: blur(8px)）跨越⻚⾯顶部。
- 所有层⾯积明显不同（⼤⾯积 A、⼩⾯积 C、全屏 O），制造多 render pass 嵌套与不同像素覆盖
  率的组合。
- 不使⽤ JS（纯静态），保证布局稳定、仅考察渲染路径。

### 人工观察与建议工具

- DevTools → Rendering：启⽤ Show composited layer borders、Paint flashing。
- DevTools → Layers ⾯板：检查是否为 A/B/C/O 创建独⽴图层与离屏缓冲；观察图层层级与合成
  顺序。

- DevTools → Performance：录制⾸帧与⼀次轻微滚动，观察 Paint 与 Composite ⽐例、GPU 内
  存峰值。

### 验收标准

- 视觉正确：混合/遮罩/模糊边缘⽆撕裂、⽆⾊带、⽆错误叠序。
- 合成合理：不同⾯积与效果的层被分配到合适数量的合成层，未出现异常巨额显存或频繁重绘。
- 滚动时，⽆明显掉帧或滤镜闪烁。

### 可选变体

- 将 filter 换为 等价 SVG filter 对⽐；
- 增⼤/减⼩模糊半径，观察离屏纹理尺⼨与光栅耗时变化；
- 去掉 isolation ⽐较合成边界变化。
