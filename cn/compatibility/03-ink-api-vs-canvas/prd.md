## 研发侧⽤例 03 ｜ Ink API 硬件墨迹合成与⽹⻚绘制对⽐

> 最⼩ JS 依赖：调⽤ Ink API；若不可⽤，优雅降级到普通 Pointer Events。

### 测试目标

对⽐ OS/合成器层⾯的硬件墨迹（低延迟、合成优先）与⽹⻚⾃⾝像素绘制（Canvas）的差异，观察
输⼊速写时的帧稳定性与重绘区域⼤⼩。

### 涉及特性

Ink API、Pointer Events、pointerrawupdate（如需）、CSS 硬件加速提⽰（will-change）。

### 页面结构与要点

- 左侧画布：Ink API 路径（若⽀持）；右侧画布：Canvas 2D 路径。
- 同步显⽰相同笔迹（粗细/透明度⼀致），背景为棋盘格或渐变以便观察混合效果。
- 计时条或节拍指⽰器做帧稳定性的⾁眼参考。

### 人工观察与建议工具

- Rendering → Paint flashing：Ink 路径理想情况下⼏乎不闪烁，Canvas 路径会频繁重绘。
- Performance：记录快写（快速连续笔划）阶段的主线程占⽤、丢帧。
- Layers：Ink 是否通过合成层合⼊，不占⽤过多光栅时间。

### 验收标准

- Ink 路径延迟更低、线条跟⼿，⼏乎不触发⼤⾯积 Paint。
- Canvas 路径可⻅重绘增加但⽆撕裂；两者视觉结果⼀致。

### 可选变体

- 透明墨迹与混合模式叠加；⾼分辨率画布（HiDPI）。
