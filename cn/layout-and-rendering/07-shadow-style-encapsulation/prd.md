测试用例 7：Shadow DOM 样式封装隔离

测试目的：验证 Chrome 对 Shadow DOM 的样式封装（Style Encapsulation）机制的支持，确保组件封装内部的元素样式与外部页面样式相互隔离 。通过此测试，可以观察自定义元素（Web Component）的 Shadow DOM 内部内容是否不受页面全局 CSS 影响，同时其内部定义的样式也不会影响页面其他部分，从而保证组件在任何环境下都保持预期的渲染。

测试特性：使用原生 Web Component（自定义元素）创建 Shadow DOM。包括：在页面中定义一个自定义元素，并在其构造函数中通过 attachShadow({ mode: 'open' })附加 Shadow Root；向 Shadow Root 插入内部 HTML 结构和对应的`<style>`定义内部样式。外部页面也定义一些全局 CSS 试图作用于自定义元素的内部节点（例如为自定义元素内部的标签设置样式），以测试这些外部样式是否被 Shadow 隔离阻断。此外，可测试 Shadow DOM 的 slot 分发机制（如在自定义元素内部使用`<slot>`投影外部提供的子节点）以及:host、::slotted 等选择器，观察样式作用范围和隔离效果。

供你参考的测试用例设计：页面中有一个自定义元素（例如`<my-element>`），它的 Shadow DOM 内部包含一个简单的 HTML 结构，如`<p class="inside">`Shadow 文本`</p>`，并在 Shadow 内的中定义.inside { color: red; }使其文本呈红色。页面全局的里也定义了.inside { color: blue; }以及尝试针对自定义元素的选择器（例如 my-element p { color: blue; }），用于测试全局样式对 Shadow 内容的影响。此外，页面上还有一个普通的`<p class="inside">`普通文本`</p>`在 Light DOM，用于对照外部样式。需要通过一段简短的 JavaScript 注册该自定义元素类并将上述结构附加到其 Shadow Root（仅用于构建测试内容，不涉及交互逻辑）。

供你参考的测试用例使用方法：打开页面后，首先检查自定义元素内部内容的样式。预期 Shadow DOM 内部的段落文本呈现为红色（来自 Shadow 内部），而非继承页面全局定义的蓝色。这证明页面 CSS 没有影响 Shadow 内部元素 。同时，页面上普通的应呈蓝色，证明全局样式正常应用于 Light DOM 元素，且 Shadow 内部的样式没有影响外部内容 。利用 Chrome DevTools 检查 DOM 结构，展开`<my-element>`可以查看其 Shadow Root 内部，验证其中样式表作用范围仅限 Shadow 内部元素。若自定义元素使用了`<slot>`插槽投影外部内容，也应验证投影内容继承的是外部样式而非 Shadow 内部样式。性能方面，由于仅涉及样式隔离，可考虑在页面上放置多个此自定义元素实例，观察样式计算是否仍各自独立无冲突，Chrome 对 Shadow DOM 样式封装的实现应保证大量组件共存时依然高效。重点是确认样式隔离的正确性，组件内部外部样式互不干扰，无意外的样式泄漏或覆盖。
