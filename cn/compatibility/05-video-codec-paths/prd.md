1. 每个测试用例都由一或多个独立网页组成，每个网页专注于服务单一的测试需求。
2. 每个网页都只能用原始的 HTML、CSS、JS、Web API 来实现，不要使用任何框架和库（比如 React、Vite、npm 等） 。

6. 代码中的注释都用英文，网页内容中的文字都用中文。
7. 每个页面都由以下元素组成：
   a. 顶栏：测试用例的标题、一句话说明

## 研发侧⽤例 05 ｜ Video 编解码与呈现路径覆盖

> 最⼩ JS 依赖：选择/切换不同编码源；可使⽤同域静态资源。

### 测试目标

在不同编码/封装（H.264/H.265/VP9/AV1 等）与分辨率/帧率下，观察硬件解码、⾊彩转换
（YUV→RGB）、缩放与 overlay 的呈现路径差异。

### 涉及特性

`<video>`、playsinline、preload；多种编码与分辨率；颜⾊空间/⾊度抽样差异（4:2:0/4:2:2）。

### 页面结构与要点

- 列表⻚展⽰若⼲ `<video>` ⼩窗（静⾳、⾃动播放、循环），覆盖多编码多分辨率。
- 详情⻚全屏播放同⼀源，测试缩放与 overlay。
- UI 仅 CSS（选择不同源）。

### 人工观察与建议工具

- DevTools → Media ⾯板：查看解码器类型（硬件/软件）、⾊彩空间。
- Performance：录制播放 10s，观察主线程负载与 GPU 合成。
- 屏幕观察：缩放时是否出现锐化/模糊异常、⾊偏或⾊带。

### 验收标准

- 能稳定硬件解码（可⽤ Media ⾯板确认），播放流畅⽆丢帧。
- 缩放/全屏时⽆⾊彩与纵横⽐错误。
- 多实例同时播放不引起明显掉帧。

### 可选变体

- HDR 源与 SDR 源对⽐（若设备/系统链路⽀持）。
- ⾼帧率（60/120fps）源的呈现稳定性。
