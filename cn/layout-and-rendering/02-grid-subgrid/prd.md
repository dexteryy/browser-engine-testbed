测试用例 2：Grid 网格与 Subgrid 子网格布局

测试目的：验证 Chrome 对 CSS Grid 网格布局及其子网格（Subgrid）功能的支持情况，确保嵌套网格能够按照父网格定义的轨道对齐。通过本测试，可以发现浏览器在多层网格布局下的排版准确性，以及子网格是否正确继承父网格的列宽/行高定义，实现跨越父子层级的对齐布局 。

测试特性：使用现代 CSS Grid 布局（如 display: grid），外层网格定义若干列和行（包括固定宽度和自适应 fr 等），并命名网格线。内层元素跨越外层网格的多个网格单元，并设置为 display: grid 启用子网格特性（例如 grid-template-columns: subgrid），从父网格继承列定义。布局中包含网格的嵌套、网格线命名、网格间隙（gap）以及媒体查询调整（例如在较窄视口下改变列数）等。

供你参考的测试用例设计：页面主体包含一个三列布局的网格容器（例如左侧栏、中间主内容区和右侧栏）。在主内容区内有一个嵌套容器元素，横跨主网格的多列范围（例如同时跨越中间和右侧栏的列线）。该嵌套容器自身设为 Grid 并启用 Subgrid，使其内部的子项可以直接使用父网格的列轨道进行对齐。比如，父网格定义了列宽比例，嵌套容器内的元素通过 Subgrid 与父级列线对齐，从而在视觉上与整个页面网格保持一致。页面可包含多个元素以测试子网格在不同场景下的表现（例如一个元素使用 Subgrid 对齐列，另一个使用 Subgrid 对齐行）。

供你参考的测试用例使用方法：首先观察页面布局是否如预期排列：嵌套网格内的元素应当准确对齐到父网格的列/行边界上，没有出现错位或无法对齐的情况（例如内层元素的边缘应与外层网格的网格线重合）。可以在 Chrome DevTools 的网格覆盖（Grid Overlay）功能中启用网格线显示，检查子网格的轨道是否与父网格重叠对齐。手动调整窗口大小或触发相关媒体查询，确认子网格布局能随父网格变化保持一致。性能方面，可留意首次渲染时网格排版耗时（对于包含大量单元格的网格，首屏绘制时间可能增加），以及在调整布局时是否有明显的延迟。确认在开启子网格特性后未出现布局异常或浏览器崩溃等兼容性问题（特别是 Subgrid 属 Chrome 新近支持功能，应验证其稳定性）。
