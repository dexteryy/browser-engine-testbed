测试用例 1：Flexbox 弹性布局与动态重排

测试目的：验证 Chrome 浏览器对 Flexbox 弹性布局的解析和渲染是否正确，包括多层嵌套的弹性容器在内容变化时的排版调整效果。特别关注当子元素尺寸动态变化（例如用户鼠标悬停导致元素扩展）时，布局重新计算（重排）是否正确且不会出现内容溢出或相互遮挡。

测试特性：使用 CSS Flexbox 布局模型（如 display: flex 容器），包含主轴对齐（justify-content）和侧轴对齐（align-items）设置；嵌套的 Flex 容器（一个 Flex item 内部再定义为 Flex 容器）；利用 CSS 伪类（如:hover）改变子元素的 flex 属性（例如 flex-grow）来模拟动态尺寸变化；不使用 JavaScript，仅通过 CSS 交互实现布局变化。

供你参考的测试用例设计：页面包含一个主容器元素，设置为 display: flex 并具有多个子项。其中一个子项本身也是 display: flex 的容器，内部再嵌套一些元素，以测试多层 Flex 布局。各子项内容高度宽度各异（如其中包含文本段落和图像），初始情况下按 Flexbox 规则平分或按指定比例分配空间。为其中一个子项添加:hover 样式，当用户将鼠标悬停其上时改变该子项的 flex 值（例如从 flex: 1 变为 flex: 2），使其占据更大空间，而其他兄弟项相应压缩。

供你参考的测试用例使用方法：在页面加载时，检查各 Flex 子元素是否按照预期对齐（例如垂直居中或顶对齐）且间距正确。然后将鼠标依次悬停在可互动的 Flex 子项上，观察其它元素是否平滑地重新布局以适应尺寸变化，确保没有内容重叠或超出容器边界的现象。可使用 Chrome DevTools 的“Layout Shift Regions”功能观察布局变化区域 （启用后重绘页面，可见发生布局偏移的区域以紫色高亮），以及 Performance 面板查看鼠标悬停触发重排时的计算开销（应只影响局部容器，而非整个页面）。确认每次重排后页面渲染仍正确无误，没有出现闪烁或断裂。
